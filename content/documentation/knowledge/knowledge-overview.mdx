### Knowledge Overview

#### What Is Knowledge in AssistCX?

Knowledge in AssistCX provides the reference information that agents use during task execution.  
It allows agents to access structured or documented information when processing requests, ensuring that outputs are accurate and aligned with organizational standards.

Knowledge collections function as a reference layer for agents. Instead of relying only on the task input or email content, agents can retrieve relevant information from linked knowledge sources when additional context is required.

By referencing knowledge, agents can follow defined policies, consult documentation, and generate responses based on trusted information rather than assumptions.

---

#### When to Use Knowledge

Knowledge should be attached to an agent when tasks require information beyond the task input itself.

This is especially useful when agents must follow defined guidelines or reference internal documentation while generating responses or making decisions.

Typical scenarios include:

- Customer support responses based on FAQs
- Company policies or operational procedures
- Product documentation and feature references
- Compliance rules or regulatory guidelines
- Internal process documentation

In these situations, linking knowledge collections helps agents produce consistent and reliable outputs.

If an agent needs additional context beyond the email or task data, attaching relevant knowledge collections improves accuracy and reduces ambiguity.
