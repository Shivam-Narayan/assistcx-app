## Settings Overview

The Settings & Admin section is the control center of AssistCX.

It provides configuration, governance, and system-level controls required to manage how the platform operates across users, agents, and workflows.

While Agents and Inbox focus on execution, Settings focus on **administration, security, and platform behavior**.

This section is primarily used by administrators and system owners.

---

### What Comes Under Settings

Settings includes all configurations that affect the overall platform rather than individual tasks.

Key areas typically include:

- Mailbox connections and polling configuration
- User roles and permissions
- Access control and visibility
- Knowledge and data templates
- Intent definitions and routing rules
- System defaults and operational parameters
- API key management for secure integrations 

These configurations determine:
- How emails are ingested
- Who can access what
- How agents behave at a system level
- How workflows are governed

In short, Settings define **how AssistCX runs behind the scenes**.

---

### Admin Responsibilities

Administrators are responsible for ensuring the platform operates securely, reliably, and efficiently.

Common responsibilities include:

#### Platform Configuration
- Set up mailbox integrations
- Configure polling and ingestion behavior
- Maintain system-level settings
- Generate and manage API keys for external integrations

#### User & Access Management
- Create and manage users
- Assign roles and permissions
- Control access to agents, tools, and knowledge
- Enforce visibility rules

#### Governance & Compliance
- Ensure proper ownership of agents and workflows
- Maintain auditability and traceability
- Manage sensitive data access

#### Operational Reliability
- Monitor system health and logs
- Resolve configuration issues
- Update integrations or credentials
- Maintain templates and intent rules

#### Continuous Improvement
- Optimize routing and automation logic
- Update configurations as workflows evolve
- Support teams with setup and troubleshooting

---

### API Keys

API Keys enable secure communication between AssistCX and external systems.

They act as authentication tokens that allow third-party applications or services to interact with AssistCX APIs without requiring manual login.

Administrators can:

- Generate API keys for integrations  
- Control and restrict access to platform capabilities  
- Revoke or regenerate keys when required  

API keys are commonly used for:

- System-to-system integrations  
- Workflow automation triggers  
- Programmatic access to platform data  

> Note: API keys should be handled securely and must not be exposed in public environments.
