## Tool Types

AssistCX supports different types of tools based on the type of action or integration required.  
Each tool type enables agents to interact with systems, execute operational actions, or perform specialized processing during task execution.

---

### Integration Tools

Integration tools connect AssistCX with external applications and services.

These tools allow agents to interact with third-party systems by retrieving or updating information and triggering actions outside the platform.

Common use cases include:

- Reading or writing data to external systems
- Triggering actions in connected platforms
- Synchronizing information across services

Examples include integrations with email platforms, storage systems, CRMs, or other business applications.

---

### System Tools

System tools are built-in tools provided by AssistCX to perform actions within the platform itself.

They support internal operational workflows and ensure that agents can manage tasks and data inside the system.

Typical use cases include:

- Updating task status
- Performing system validations
- Managing internal data operations
- Supporting execution utilities required during workflows

These tools ensure that agents can perform necessary platform-level operations without relying on external integrations.

---

### API or Custom Tools

API or custom tools allow teams to create tools tailored to specific business needs.

These tools enable agents to execute custom logic or interact with systems that are not supported through standard integrations.

Common use cases include:

- Calling internal or external APIs
- Implementing custom business logic
- Performing specialized data processing
- Integrating with proprietary systems

Custom tools provide flexibility for advanced workflows and allow organizations to extend AssistCX to fit their operational environment.
