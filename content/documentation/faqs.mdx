### Frequently Asked Questions

---

##### Do I need to configure everything before using AssistCX?

No. You can start by connecting a mailbox and creating an agent for a specific workflow. Additional configuration such as tools, knowledge sources, and planning steps can be added gradually as the workflow evolves.

---

##### What happens if an email cannot be processed automatically?

If the system cannot process an email correctly, the task may remain incomplete or produce an unexpected result. In such cases, users can review the task in the Task Inbox, retry execution, update the configuration, or report an issue for investigation.

---

##### Can multiple agents work on different workflows?

Yes. Different agents can be configured for different operational workflows. For example, one agent may handle support requests while another processes invoices or approvals.

---

##### Can users manually intervene in automated workflows?

Yes. Users can review tasks, update task status, retry execution, add notes, or report issues when manual intervention is required. This ensures that automation remains flexible and supervised when necessary.

---

##### What happens if external systems or integrations fail?

If an external integration fails during execution, the task may produce an error or incomplete output. Users can review execution logs, retry the task once the system becomes available, or investigate the issue through the Issue Management module.

---

##### How does AssistCX maintain visibility into automated workflows?

All emails, tasks, outputs, logs, and issues are recorded within the platform. This allows teams to monitor workflows, track execution results, and review operational history directly from the Task Inbox and related modules.

---

#### Can workflows be improved over time?

Yes. As teams identify issues, adjust configurations, or update knowledge sources, agents can be refined to improve accuracy and reliability. This allows workflows to evolve as operational requirements change.
