### Task View

Task View displays the tasks generated from emails and provides the workspace for reviewing execution results.

When a task is selected, the workspace shows the task content, agent output, and execution details.

This view allows users to inspect how the task was processed and take action when required.

---

### Task Information

Each task contains key information required to understand the work being processed.

Common task details include:

- **Task ID** – Unique identifier for the task  
- **Status** – Current execution state (such as Incomplete or Completed)  
- **Source email** – The email from which the task was created  
- **Assigned agent** – The agent responsible for processing the task  
- **Timestamps** – When the task was created and completed

These details help users quickly identify the task and track its progress.

---

### Execution Results

The task workspace displays the output produced by the agent.

This typically includes:

- **Task Status**  
- **Final Answer** – The agent’s generated result  
- **Task Summary** – A concise explanation of the work performed

Users can also switch to **JSON view** when structured output is available.

---

### Execution Log

The **Execution Log** records how the agent processed the task.

Logs may include:

- Task input information  
- Intermediate processing steps  
- Tool invocations  
- Validation checks  
- Final output generation

This log provides transparency into the agent’s execution process and helps with troubleshooting.

---

### Task Overview Panel

Selecting **Task Overview** opens a sidebar with additional task context.

This panel provides:

- Task metadata  
- Description and attachments  
- Tags  
- Assigned agent details  
- Agent instructions and rules

The sidebar allows users to review task information without leaving the execution view.

---

### Task Actions

From the task actions menu, users can perform operational actions such as:

- **Retry Task**
- **Change Status**
- **Archive Task**
- **Report Issues**
- **View Email**
- **View Activity**

These actions allow users to intervene when execution requires manual review.

---

### When to Use Task View

Task View is used when users need to:

- Review agent execution results
- Inspect logs and processing steps
- Investigate failed or incomplete tasks
- Take corrective actions on a task

This view provides the operational workspace for managing task execution and resolution.
