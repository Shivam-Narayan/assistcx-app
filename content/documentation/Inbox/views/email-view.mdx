### Email View

Email View displays all incoming emails captured from connected mailboxes before or during task processing.

Each email appears as an item in the inbox list and provides the context used to generate tasks.

Selecting an email loads its details and associated tasks in the workspace.

---

### What You Can See

Each email entry includes key information needed to understand the request:

- **Sender** – The person or system that sent the email  
- **Subject** – The email subject line  
- **Timestamp** – When the email was received  
- **Preview text** – A short excerpt from the message  
- **Attachments indicator** – Shows if files are included  
- **Task status** – Indicates whether tasks exist and their current state

This allows users to quickly identify which emails require attention.

---

### Email Details

When an email is selected, the workspace displays additional information including:

- Email body content  
- Attached files  
- Associated tags  
- Tasks generated from the email

This context helps users understand the request before reviewing or executing tasks.

---

### Creating Tasks from Emails

Emails can be converted into tasks so they can be processed by agents.

Task creation may occur in two ways:

**Automatic Task Creation**  
Tasks may be generated automatically when predefined rules or workflows detect actionable content.

**Manual Task Creation**  
Users can create tasks manually when an email requires review before automation begins.

---

### When to Use Email View

Email View is useful when users need to:

- Review incoming emails before automation begins
- Understand the context behind generated tasks
- Inspect attachments or message content
- Manually create tasks when required

This view provides the entry point for work that begins from email communication.
