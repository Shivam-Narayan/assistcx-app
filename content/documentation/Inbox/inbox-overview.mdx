### Inbox Overview

The **Task Inbox** is the primary workspace where incoming emails are converted into tasks and processed through agents.

Instead of working directly from an email client, AssistCX organizes incoming messages into structured tasks. This allows teams to track work, monitor execution, and review results in a single interface.

Each item in the inbox represents an email that may contain one or more tasks generated for processing.

---

### How the Task Inbox Is Organized

The Task Inbox interface is divided into three main areas:

**Email List**  
Displays all incoming emails from connected mailboxes.  
Each entry shows the sender, subject, timestamp, and task status.

Selecting an email loads its details and associated tasks.

---

**Task Workspace**  
Shows the content of the selected email and the tasks generated from it.

This workspace includes:

- Email information
- Attachments and tags
- Generated tasks
- Agent execution output
- Execution logs

Users can review how the task was processed and inspect the final output generated by the assigned agent.

---

**Task Overview**  
Provides additional context about the selected task.

This panel includes:

- Task metadata
- Creation and completion timestamps
- Attachments and tags
- Assigned agent details

The Task Overview panel can be opened without leaving the task workspace, allowing users to inspect task information while reviewing execution results.

---

### Task Lifecycle in the Inbox

Each item in the inbox typically follows a structured lifecycle:

**Email Received**  
Incoming messages are captured from configured mailboxes.

**Task Generated**  
The system creates one or more tasks from the email.

**Agent Execution**  
The assigned agent processes the task using its configured instructions, tools, and planning logic.

**Output Generated**  
Execution results appear in the Final Output section.

**Review or Intervention**  
Users can review results, retry execution, update task status, or take additional actions when required.
